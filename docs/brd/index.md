# 量潮课堂 - 业务需求文档

## 场景一：学生分类与权限管理

### 工作角色

- 运营方：管理各班级和训练营的学员
- 学生：报名、参与课程和实训

### 问题

量潮课堂有多种学生类型，不同类型权限不同。

触发：学生报名时需要确认身份类型。

现状：付费学员免审核购买课程和实训，VIP 学员免审核任何课程，免费学员需要通过选拔和阶段性考核。

困难：运营方需要人工识别学生类型，手动分配权限，人数多了容易出错。

后果：权限分配错误导致学生无法正常使用或白嫖课程，运营方需要反复处理投诉。

### 假设

如果存在这样一个平台，能够根据学生类型自动匹配权限，自动化选拔和考核流程，运营方就能高效管理学生。

## 场景二：课程体系管理

### 工作角色

- 运营方：管理课程发布、班级和训练营
- 学生：选课和学习

### 问题

量潮课堂有理论课和实践课两个系统，课程体系复杂。

触发：学期开始或新课程上线时，运营方需要创建班级或训练营。

现状：理论课有微专业和班级（浙理班级、社会招生班级），实践课有实训基地和训练营（大数据训练营、CEO助理训练营、VIP训练营），规则各不相同。

困难：不同类型课程有不同的管理规则，运营方需要管理大量课程和班级，人工维护成本高。

后果：规则不统一导致管理混乱，学生报错名，运营方疲于应对。

### 假设

如果存在这样一个平台，能够统一管理不同类型的课程和班级，自动处理报名、选拔、考核等流程，就能降低管理成本。

## 场景三：多渠道招生管理

### 工作角色

- 运营方：管理招生流程
- 学生：报名

### 问题

量潮课堂通过社会招生和校企合作两种方式招生，规则不同。

触发：学生报名时，运营方需要确认招生渠道。

现状：社会招生参考人力资源管理办法，校企合作根据协议免审核。

困难：不同渠道有不同的规则，运营方需要分别处理，容易混淆。

后果：规则用错导致招生出错，学生投诉，影响营收。

### 假设

如果存在这样一个平台，能够统一管理不同招生渠道，自动匹配对应的规则，就能简化招生流程。

## 场景四：多方协作与权责界定

### 工作角色

- 事业部：负责营收、人才供给、交付供给
- 运营团队：管理各班级和训练营，与学员协商制定工作章程
- 秘书处：复核工作

### 问题

量潮课堂涉及多方协作，职责需要明确。

触发：需要制定或修改工作章程时，各方需要对齐。

现状：事业部负责运营，秘书处复核，运营团队执行。信息分散在不同人手里。

困难：多方协作时，信息不同步，职责不清，遇到问题相互推诿。

后果：工作章程执行不到位，出了问题没人负责，影响团队信任。

### 假设

如果存在这样一个平台，能够明确各方职责，沉淀工作章程，追踪执行情况，多方协作就能更顺畅。
