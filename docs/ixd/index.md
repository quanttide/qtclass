# 量潮课堂交互设计

聚焦「上课」场景，描述学生和教师在一次授课中的完整交互。

## 用户目标

- 学生：准时进入课堂，跟随授课进度，获取学习材料
- 教师：按计划授课，管理课堂秩序，记录授课情况
- 运营方：跟踪课堂状态，统计出勤和课时

## 上课流程

1. 上课前：学生和教师各自查看课表，确认今日课程安排
2. 课前准备：教师进入课堂，查看学员名单和课程材料
3. 授课进行：教师按课次推进，学生跟随进度
4. 下课：教师结束课堂，系统记录出勤和授课时长

## 页面

### 课表

用户看到的首页，展示当日及本周的课程安排。

按时间线排列，每项展示课程名称、班级、时间、地点（线上/线下）、授课教师。

点击某节课进入授课详情。

交互规则
- 正在进行中的课程置顶高亮，显示「进入课堂」按钮
- 已结束的课程灰显，可点击查看回顾
- 今日无课时显示空状态引导

### 课堂

一次授课的核心页面，分为上课中和课后回顾两种状态。

上课中
- 顶部：课程标题、班级、当前时间进度条
- 主体：内容区（课件/白板/共享屏幕）、侧边学员列表
- 工具栏：出勤标记、互动工具（提问/投票）、笔记
- 底部控制：下课按钮（仅教师可见）

课后回顾
- 出勤统计：应到/实到/缺勤
- 授课记录：实际授课时长、完成课次
- 材料下载：课件和补充资料列表
- 操作：编辑授课备注

### 工具栏交互

上课中教师的操作工具集中展示。

出勤标记
- 点击学员头像标记出勤状态（正常/迟到/缺勤）
- 支持批量标记（全勤/全部迟到/全部缺勤）
- 标记后学员头像显示状态标识

互动工具
- 提问：向全体发送提问，学员回复后展示结果
- 投票：发起快速投票，结果实时展示

授课备注
- 教师记录当次课的补充说明
- 支持文本输入和附件上传
- 下课后自动保存

应用底部三个页签，但本次只实现第一个：

| 页签 | 说明 |
|------|------|
| 上课 | 课表 + 课堂（本次范围） |
| 课程 | 延后实现 |
| 我的 | 延后实现 |

课表和课堂之间使用堆叠导航：课表中选择课程→滑入课堂页，返回时课堂收起。
